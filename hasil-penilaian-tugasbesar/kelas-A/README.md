### Rubrik Individu (untuk C03, C06, C13)

| Komponen | Bobot | Penjelasan |
|---|:---:|---|
| Konsistensi | 30% | Berapa minggu dari total minggu fase yang anggota aktif commit |
| Substansi | 40% | Volume commit yang substantif (bukan typo/merge) + kualitas output |
| Kesesuaian Peran | 20% | Apakah commit sesuai role (Backend/Frontend/DevOps/QA & Docs/dll) |
| Kualitas Pesan | 10% | Deskriptif, conventional commits, ada konteks |

### Baseline Floor 70 (Role-Aware)

Anggota dengan role yang bukan prime-time untuk fase tertentu (misalnya DevOps di Fase 1 Foundation) cenderung punya volume commit rendah karena memang bukan tugas utamanya. Untuk mengakomodasi ini, diterapkan **baseline floor 70** untuk anggota off-phase yang masih punya kontribusi minimal (skor mentah ≥ 50). Anggota prime-time tetap dinilai ketat tanpa floor; anggota yang benar-benar absen total (skor < 50) juga tidak mendapat floor.

**Pemetaan prime time per fase:**

| Fase | Prime time roles | Off-phase roles |
|---|---|---|
| **F1 Foundation** | Backend, Frontend, QA & Docs | DevOps, Container, CI/CD, Deploy |
| **F2 Containerization** | DevOps, Container, CI/CD, Deploy, Backend | Frontend, QA & Docs |
| **F4 CI/CD** | CI/CD, DevOps, Deploy, QA & Docs | Backend, Frontend |

**Formula:**
```
adjusted_score = max(raw_score, 70)   jika role off-phase AND raw_score ≥ 50
adjusted_score = raw_score            jika role prime-time atau raw_score < 50
```

Skor yang ditampilkan di tabel master sudah post-baseline. Skor mentah ada di file laporan kelompok masing-masing (Section C).

### Kolom Adjustment (Manual Dosen)

Kolom **`Adj`** di tabel master kosong, untuk dosen isi manual berdasarkan observasi yang tidak terlihat di repo:
- Partisipasi diskusi kelas / presentasi
- Kontribusi non-commit (review, design, testing manual)
- Konteks magang / situasi khusus
- Penalti untuk free-riding yang tidak terdeteksi di git log

Range yang disarankan: **±10 poin**. Hasil muncul di kolom **`Total Adj`** = Total + Adjustment.

### Keterbatasan

1. Commit count adalah proxy, bukan ukuran sempurna. Pair programming, refactor besar (1 commit besar), code review, dan presentasi tidak terlihat di git log — kolom `Adj` ada untuk koreksi.
2. Squash merge bisa membuat 1 anggota terlihat dominan padahal kontribusi tim. Pattern PR ikut dicek untuk verifikasi.
3. Alias git name sudah dikonsolidasi (mis. `Hahahanaa` = Zahwa, `intniaaa20` = Debora) via email matching.
4. Magang/konversi dinilai dengan rubrik sama tapi di tabel terpisah karena NIM tidak ada di gradebook reguler xlsx.
